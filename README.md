# GitHub Collaboration Workflow

## Overview

This repository documents a practical GitHub collaboration workflow completed as part of an open-source contribution exercise.

The project demonstrates how to collaborate using Git and GitHub by working with a forked repository, creating feature branches, making documentation contributions, and submitting Pull Requests for review.

The objective was to follow a real-world collaborative development workflow rather than simply practice Git commands.

## Skills Demonstrated

- Forking an existing GitHub repository
- Cloning a remote repository locally
- Configuring `origin` and `upstream` remotes
- Synchronizing a fork with the upstream repository
- Creating and managing feature branches
- Making documentation contributions
- Writing meaningful commit messages using Conventional Commits
- Pushing changes to GitHub
- Creating and submitting Pull Requests
- Collaborating using the standard GitHub contribution workflow

## Evidence of Contribution

This workflow resulted in multiple contributions to an open-source project, including:

- Added my contributor profile to the project's `CONTRIBUTORS.md`.
- Improved the project's documentation by clarifying browser prerequisites in the `README.md`.
- Submitted Pull Requests following the standard GitHub collaboration workflow.
